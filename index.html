    Como aplicar no GitHub Web
    1. Baixe o trips.json com o botão acima.
    2. Abra data/trips.json no GitHub Web e clique no ícone do lápis (canto superior direito).
    3. Selecione tudo (Ctrl+A) e cole o conteúdo do arquivo baixado (Ctrl+V).
    4. Commit (mensagem "edições do site — DATA" serve).
    5. Volte aqui e clique "Descartar edições locais" — elas já estão no repo.

    ⚠ Não esqueça do passo 5. Senão as edições locais e do repo vão divergir.

    ❓ Como usar este site

    🌍 O que é este site

    Este é o portfólio pessoal de viagens do Eduardo Campos. Aqui ficam registradas todas as viagens realizadas desde 2021, as que estão planejadas, e as que ainda são sonhos na wishlist. O site funciona como diário visual e ferramenta de planejamento das próximas aventuras.

    📖 Glossário de termos e ícones

    Termo/ÍconeSignificado
    ✓ RealizadasViagens já concluídas
    📅 PlanejadasViagens com data e reservas confirmadas
    ⭐ WishlistDestinos desejados, ainda sem data
    Pin colorido (mapa)Viagem realizada
    Pin tracejado (mapa)Viagem planejada ou wishlist
    🧳 BagagemAgente que monta lista de bagagem
    💡 InspiraçãoAgente que sugere destinos
    ⚖ CompararComparador de viagens lado a lado

    🧭 Fluxo recomendado de uso

    • Antes da viagem → use 💡 Inspiração para escolher destino e ⚖ Comparar para decidir entre opções.
    • Durante o planejamento → preencha Checklist, Reservas, Orçamento e Notas na aba do destino.
    • 1-2 semanas antes → ative 🧳 Bagagem para montar a mala sem esquecer nada.
    • Depois da viagem → atualize Notas & Plano para virar memória permanente.

    👤 Quem é Eduardo

    Eduardo Campos viaja desde 2021 documentando cada destino aqui. Este site funciona como diário público e ferramenta de planejamento das próximas aventuras, das memórias mais antigas aos sonhos ainda na wishlist.

    💬 Perguntas frequentes

    Posso editar as viagens?
    Não, o site é público mas só o Eduardo edita o conteúdo.
    Como sugerir um destino?
    Mande mensagem para o Eduardo pelos canais habituais de contato.
    Funciona offline?
    Sim — depois da primeira visita, o conteúdo fica em cache e o site continua acessível sem internet.
